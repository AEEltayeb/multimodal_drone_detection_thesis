Yes — and the best way to start is not a new end-to-end fused detector. Since you already have an RGB detector and an IR detector, the lowest-risk path is late fusion: run both models, then train a small meta-classifier on their outputs. In multispectral detection, fusion is commonly grouped into early, feature-level, and late/decision-level fusion; there are also strong late-fusion baselines for RGB-thermal detection such as Bayesian Fusion and box-fusion methods. 
A “classifier on top” is basically stacked generalization: a second model takes the base models’ outputs as input and learns when to trust RGB, IR, or both. A useful result from the stacking literature is that the meta-model should use the confidence outputs, not just hard yes/no predictions. 
What I would recommend
Best first version: detection-level late fusion
This is the version to build first if your RGB and IR frames are synchronized.
Run both detectors on each paired RGB/IR frame. 
Bring detections into a common coordinate frame. 
If the cameras are aligned, use the boxes directly. 
If not, project one modality into the other using calibration or a homography. 
Form candidates from the union of RGB and IR boxes. 
Match RGB and IR boxes that likely refer to the same object. 
Train a small classifier on each candidate to predict: real drone or false alarm. 
Use that classifier’s probability as the final fused score. 
For the final box coordinates, either keep the better box or fuse matched boxes with Weighted Boxes Fusion. WBF explicitly averages boxes using their confidence scores instead of just suppressing them. 
This is usually much easier and more data-efficient than retraining a dual-stream RGB-IR network from scratch.
What features should the top classifier use?
For each candidate detection, I’d start with:
p_rgb: RGB detector confidence 
p_ir: IR detector confidence 
only_rgb, only_ir, both_present 
IoU between matched RGB and IR boxes 
normalized box center/width/height 
area of the box 
per-modality quality cues: 
RGB brightness / blur 
IR contrast / thermal saliency 
optional temporal cues: 
confidence averaged over last 3–5 frames 
whether the same candidate existed in previous frames 
The key idea is: the classifier should learn patterns like:
“trust IR more at night” 
“trust RGB more in clear daylight” 
“trust both most when both agree spatially” 
“downweight RGB when blur is high” 
“downweight IR when thermal contrast is weak” 
What classifier should sit on top?
Start simple:
Logistic regression first 
then XGBoost / LightGBM if needed 
I would not start with a neural net on top unless you have a lot of paired data. A small tabular classifier is easier to debug and usually enough for a first fusion system.
How do you make the training set for the fusion classifier?
This is the critical part.
For each frame:
collect all candidate detections from the RGB+IR union 
match each candidate to ground truth 
label it positive if it overlaps a real drone enough, for example IoU >= 0.5 
otherwise label it negative 
That gives you a tabular dataset like:
[p_rgb, p_ir, iou_pair, only_rgb, only_ir, area, blur_rgb, contrast_ir, ...] -> {0,1}
Then train the classifier on that.
Very important: train this fusion classifier on held-out predictions, not on the same frames the base detectors were trained on. In practice, use a validation fold or out-of-fold predictions from your RGB/IR detectors. Otherwise the top classifier will overfit to overconfident base-model outputs.
How do you get the final fused box?
Use one of these:
Easy: keep the box from the modality with higher confidence 
Better: if RGB and IR boxes match, use WBF 
Very simple baseline: average the coordinates of matched boxes, weighted by confidence 
If only one modality fires, keep that box and let the classifier decide whether to accept it.
A very strong baseline to beat
Before training any classifier, compare against these baselines:
RGB only 
IR only 
OR fusion: keep a detection if either model fires 
AND fusion: keep only if both fire 
Score averaging on matched boxes 
WBF 
Bayesian Fusion 
Bayesian Fusion is a good non-learned late-fusion baseline for RGB-thermal detection, and it was proposed specifically to combine detections from different modalities at the box level. 
If your learned classifier does not beat OR, WBF, or Bayesian Fusion, then the classifier is not adding real value yet.
If you want an even simpler “classifier on top”
There’s an even easier version: scene-conditioned gating.
Instead of classifying each candidate box, train a small classifier to predict scene conditions like:
day / night 
clear / hazy 
high blur / low blur 
high IR contrast / low IR contrast 
Then use it to set weights:
score_fused = w_rgb(scene) * p_rgb + w_ir(scene) * p_ir
This is simpler than per-box stacking and is close in spirit to recent scene-specific and condition-aware fusion work, where a lightweight classifier or condition cue adapts each modality’s contribution at inference time. 
My recommended build order
If I were doing this project, I’d do it in this order:
OR fusion baseline 
Matched-box score averaging 
WBF 
Logistic-regression meta-classifier 
XGBoost meta-classifier 
Optional: scene-conditioned weighting 
Only after that: end-to-end RGB-IR fused detector 
That sequence tells you quickly whether fusion is helping because of:
complementary modalities, 
better box merging, 
or smarter trust weighting.

One important fork
If your RGB and IR cameras are not synchronized/aligned, then don’t start with box-level fusion. Start with a frame-level classifier instead:
max RGB confidence 
max IR confidence 
number of detections in each modality 
top box size 
temporal consistency 
and predict “drone present in this time window” first. Box-level fusion only becomes clean once the geometry is handled.
My honest recommendation
For your setup, the highest-probability first success is:
late fusion 
box matching 
logistic regression on detector confidences + agreement features 
WBF for matched boxes 
That gives you something interpretable, cheap to train, and easy to compare against simple OR fusion.
