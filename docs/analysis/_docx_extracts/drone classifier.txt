Svanstrom (28.7K frames, 11.7K with drone) — THE STRESS TEST
WARNING
RGB is catastrophically bad on Svanstrom: 5.4% precision, 7.6% recall. It fires 16,481 times but only 884 are correct. The OR gate inherits this garbage — precision drops to 53%. This is exactly why naive fusion fails and why an ML-based approach matters: the classifier learns "on Svanstrom-like scenes, don't trust RGB."

All 10 approaches evaluated on the same test set (45K frames, sequence-level split).
Final Ranking
Per-Class Trust Performance (Approach #10)
trust_rgb is the hardest class (only 3.9% of data, 83.3% recall) — this is the minority case where only RGB detected correctly and IR missed. The classifier catches 83% of these.

Key Findings
1. FN Models Add Zero Value
The 6 FN model features are completely redundant. The classifier learns equivalent patterns directly from raw scene features.

3. Svanstrom Reveals Why Fusion Matters
Without fusion ML, the OR gate would fire on 16K RGB false detections, crashing precision to 53%. The classifier learns to suppress RGB on Svanstrom-like scenes.
4. Detection Recall is Near-Perfect
The fusion classifier (#10) misses only ~0.02% of drones (vs 5.5% for IR alone, 13.9% for RGB alone). This is the operational win: you almost never miss a drone.

No dependency on pre-trained failure models
Simpler architecture, equivalent performance

A straightforward XGBoost classifier using detection statistics and scene features achieved 99.3% detection precision and 99.98% recall on 152K paired frames — surpassing both individual modalities and all alternative fusion strategies. Pre-trained failure-prediction models were found to contribute no incremental value, as the fusion classifier learns equivalent failure patterns directly from raw features."

Key Findings

Unchanged conclusion — the FN model features contribute nothing.
2. Consensus + Solo ML (#08) is Now Competitive
With corrected matching, the gap between #08 and #04/#10 shrank from 4.8 points to 1.9 points:
Before IoP fix: #04=0.950, #08=0.903 (gap=0.047)
After IoP fix: #04=0.967, #08=0.965 (gap=0.019)
This is because the fix moved ~6,700 Svanstrom frames from trust_ir to trust_both — and #08's hard rule "both_detect → trust_both" is now correct much more often.
3. Correcting GT Labels Was Critical
The IoP matching fix changed 7,094 rows (4.7% of data), almost entirely on Svanstrom. This raised ALL approaches' metrics by 1-6 points and changed the narrative from "clever architectures fail" to "they're competitive when data labels are corrected."
4. Missing Svanstrom GT Labels Remain
Some Svanstrom RGB frames have drones visible but no GT annotation. This still underestimates RGB precision. The caveat should be noted in the thesis.

Practical Recommendation
Approach #10 (Plain 4-class XGBoost, 40 features, no FN models) remains the winner:
98.7% trust accuracy, 96.7% F1-macro
99.75% detection precision, 99.99% recall (3 missed, 98 FP out of 45K frames)
No dependency on pre-trained failure models
Simpler architecture
Approach #08 (Consensus + Solo ML) is a strong alternative for interpretability — it uses hard rules for the easy 87% of frames and only invokes ML for the 13% disagreement cases.
Thesis Narrative
"We evaluated 10 fusion architectures on 152K paired RGB+IR frames using IoU and containment matching to handle label quality variations. A 4-class XGBoost classifier using detection statistics and scene features achieved 98.7% trust accuracy and 99.99% detection recall. Three architectures (full classifier, plain classifier, consensus+ML) achieved near-identical performance, indicating the fusion problem is well-constrained. Pre-trained failure models added no incremental value."

F1m: 
F1 macro — the average F1 score across all 4 classes (reject_both, trust_rgb, trust_ir, trust_both), treating each class equally regardless of size.
It matters because trust_both is 75% of the data, so plain accuracy gets inflated by just guessing that. F1 macro forces the model to perform well on the rare classes too — especially trust_rgb (5.3%) and trust_ir (7%).

N_test:
Number of test frames with that label. Out of the 45,088 test frames:
reject_both: 5,670 frames (12.6%) — no drone or both missed
trust_rgb: 2,370 frames (5.3%) — only RGB caught the drone
trust_ir: 3,170 frames (7.0%) — only IR caught the drone
trust_both: 33,878 frames (75.1%) — both caught the drone
trust_rgb being the smallest class (2,370) is why it has the lowest F1 — less training data to learn from.

import joblib
import numpy as np
# Load
bundle = joblib.load("fusion_no_fn_model.joblib")
model = bundle["model"]       # XGBClassifier
features = bundle["features"] # list of 40 feature names
# Predict (given a feature dict for one frame)
X = np.array([[frame_features[f] for f in features]])
trust_label = model.predict(X)[0]
# 0 = reject_both, 1 = trust_rgb, 2 = trust_ir, 3 = trust_both
# With probabilities
probs = model.predict_proba(X)[0]
# probs[0] = P(reject), probs[3] = P(trust_both), etc.
