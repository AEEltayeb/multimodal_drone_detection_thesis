  "C++" — what it stands for

  Just my naming for the ablation series:
  - Lever C = drop the 6 detection flags (ir_detected, rgb_detected, 
  both_detect, neither_detect, rgb_only_detect, ir_only_detect)      
  - C++ = Lever C plus more — also drops the 2 detection counts      
  (ir_n_dets, rgb_n_dets)

  In short: C dropped the binary "did anything fire" signals; C++    
  dropped those and the count form, which is just the same signal in 
  another encoding.

  Why the 32-feature classifier is better than the 40-feature one    

  Same accuracy, but no leakage shortcut.

  The trust label (reject_both / trust_rgb / trust_ir / trust_both)  
  is derived from per-modality TPs in build_fusion_dataset.py:       
  - A modality has TPs → it has detections → its *_detected flag is 1   → its *_n_dets ≥ 1.

  So those 8 features are near-tautological with the label by        
  construction. XGBoost discovered this and built a 35%-importance   
  shortcut on ir_detected. The model wasn't really learning — it was 
  rediscovering the labeling rule.

  When you remove those 8 features, the classifier has to predict    
  trust from things that don't directly determine the label: box     
  geometry, confidence, position, image content. And it does — at the   same accuracy. That means:

  1. Honest signal: every prediction comes from a real visual        
  feature, not a leak.
  2. OOD robustness: on data IR YOLO never saw (e.g. VTUAV with 42%  
  IR FP rate), the 40-feat model would blindly trust IR firings. The 
  32-feat model has no such reflex — it asks "does this look like a  
  drone shape, with drone-like confidence, in a drone-like position?"   first.
  3. Defensible architecture: in your thesis, "I trained an XGBoost  
  on 40 features and it got 98%" sounds different from "I removed 8  
  features that leak the label and it still got 98% because the box  
  geometry + confidence carry the real signal."
